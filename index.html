How it works

Four steps, no learning curve

It lives in the right-click menu you already use. No dashboard, no account, no cloud.

01

Right-click

Pick Lock with FolderVault on any folder in Explorer.

02

Set a password

The folder is encrypted into one .fvlt file with a padlock icon.

03

Double-click

Opens a clean unlock prompt. Right password → the folder is back.

04

Stay safe

3 wrong tries → 24-hour lockout. A recovery code always gets you in.

Features

Small, native, and honest

Built in Rust on raw Win32 — no .NET, no Electron, no webview.

Real encryption

AES-256-GCM content encryption with Argon2id password hashing. Filenames are encrypted too — a locked folder is one opaque file.

Master recovery code

Shown once on first run. Unlocks anything even if you forget a password. Only its public half is ever stored on disk.

Brute-force lockout

Three wrong passwords lock that folder for 24 hours. The counter is tamper-evident; the recovery code still works.

Crash-safe

Lock and unlock stage, fsync, then rename — an interrupted operation is journalled and recovers on next launch. Never loses data.

Tiny & quiet

A ~360 KB executable, ~190 MB/s encryption, and zero background processes. It runs only while a dialog is open.

Explorer-native

Right-click to lock, double-click to unlock, a folder-with-padlock icon, and a top-level Windows 11 menu entry.

🔒 Honest about the limits

Encryption protects the secrecy of your locked folder — not its existence. A .fvlt file is still your file on your disk, so it can be deleted like any other. The optional password-to-delete prompt is a convenience, not a hard block, and the tool tells you so. The only real protection against losing data is a backup. No backdoors: a forgotten password and lost recovery code means the data is gone by design.
